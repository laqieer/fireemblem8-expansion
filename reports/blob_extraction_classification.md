## Wave 0 harness findings

- `scripts/verify_blob_extraction.py` provides two gates:
  - `wrap LABEL SIZE` — adds a `.L_end/.if size != SIZE/.error` assertion that
    travels with the data, so a wrong length fails the build *at that blob*.
  - `check fragment.s original.bin` — assembles the fragment standalone, objcopies
    to raw bytes, and `cmp`s against the original. Reports the first differing
    offset.
- **Validated finding:** running `apdump.py` on `SpriteAnim_ArrowTrap.bin` then
  `check` reports the generated data is **4 bytes short** — the original ends in
  `00 00 ff ff` (anim-list terminator / padding) that `apdump.py`'s heuristic
  table sizing does not emit. The AP-definition converter (Wave 4) must append the
  exact structural trailing bytes; the size assertion + `check` gate enforce it.
  This confirms the harness catches the precise heuristic miss flagged in design
  review, rather than letting it slip through as a silent ROM mismatch.

## Verification gates (every converted blob)

1. Classifier invariant holds for the chosen format.
2. `verify_blob_extraction.py check` → byte-identical (exact length).
3. In-assembly `.if size != EXPECTED / .error` guard present.
4. `make -j$(nproc) compare` → `fireemblem8.gba: OK`.
5. Map-address diff: converted label and the next symbol unchanged.
